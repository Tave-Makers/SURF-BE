# 신규 팀원 인계 계획

## 개요

| 항목 | 내용 |
|------|------|
| 대상 | 신규 팀원 |
| 방식 | 1단계: 문서화 → 2단계: 대면 코드 워크스루 |
| 핵심 | admin 기능 (score, badge, member, activity) 중점 |

---

## 1단계: 문서화

총 3개 문서 작성 예정

### 문서 1. `docs/setup.md` — 로컬 환경 세팅 가이드

| 항목 | 내용 |
|------|------|
| 사전 설치 | JDK 17, MySQL, Redis |
| `.env` 구성 | 변수 목록 + 각 값 설명 (카카오 앱키, DB 접속 정보 등) |
| 카카오 개발 앱 | 로컬용 redirect URI 등록 방법 |
| 빌드/실행 | `./gradlew bootRun` + Swagger 확인 경로 |
| 자주 겪는 오류 | Redis 연결 실패, 토큰 만료 등 |

### 문서 2. `docs/architecture.md` — 아키텍처 & 패턴 가이드

| 항목 | 내용 |
|------|------|
| 레이어 구조 | Controller → Usecase → Service → Repository 흐름 |
| 네이밍 컨벤션 | CLAUDE.md 내용 기반, 실제 코드 예시 포함 |
| Usecase 패턴 | 왜 쓰는지 + score/badge 실제 코드 링크 |
| Event 패턴 | 알림 발송 흐름 (comment → notification) |
| API 경로 규칙 | `/v1/user/`, `/v1/admin/`, `/v1/manager/` 분리 이유 |

### 문서 3. `docs/domains.md` — 핵심 도메인 개요

admin 중심으로 상세하게, 나머지는 간략히

| 도메인 | 문서 수준 | 포함 내용 |
|--------|-----------|-----------|
| `score` | **상세** | 점수 부여 조건, 이벤트 트리거 시점, 2기 개선 예정 사항 |
| `badge` | **상세** | 배지 조건 로직, usecase 패턴, 현재 한계점 |
| `member` | **상세** | 회원 상태/권한 구조, admin이 다룰 수 있는 범위 |
| `activity` | **상세** | activeGeneration / activityRecord 분리 이유, 흐름 |
| 나머지 | 간략 | 한 줄 설명 + 주요 API 경로만 |

## 2단계: 코드 워크스루 세션 (대면, 약 2.5~3시간)

### 세션 1 — 전체 구조 오리엔테이션 (30분)

- `global/` 훑기: Security 설정, JWT 필터 흐름, 공통 응답 구조
- 인증 흐름: 카카오 로그인 → JWT 발급 → API 호출 → 권한 체크
- 네이밍 / 레이어 컨벤션 (왜 이렇게 나눴는지)

### 세션 2 — 단순 도메인으로 전체 흐름 실습 (30분)

`board` 도메인으로 Controller → Service → Repository 한 사이클 직접 추적

- 코드 보면서 실제 request/response 흐름 따라가기
- Swagger에서 직접 API 호출해보기

### 세션 3 — admin 핵심 도메인 워크스루 (1시간 20분)

> 2기 보완 예정이므로 **현재 구현 상태 + 한계점 + 개선 방향**을 함께 설명

| 순서 | 도메인 | 시간 | 집중 포인트 |
|------|--------|------|-------------|
| 1 | `member` | 20분 | 회원 상태/권한 구조, admin이 제어할 수 있는 범위 |
| 2 | `activity` | 20분 | activeGeneration(현재 기수 설정) / activityRecord 분리 구조, usecase 흐름 |
| 3 | `score` | 20분 | 점수 부여 트리거 (어떤 행동이 점수를 주는지), Event 연동 방식 |
| 4 | `badge` | 20분 | 배지 조건 로직, score와의 연동, 현재 한계 + 2기 개선 방향 |

### 세션 4 — Q&A + 개선 방향 공유 (20분)

- 팀원 질문
- 2기에서 보완할 admin 기능 방향 공유
- 코드 작업 시 주의사항 (타 도메인 Repository 직접 참조 금지 등)
