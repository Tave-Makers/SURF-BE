## 특정 회원의 활동기록 조회

### 개요

특정 회원의 활동기록을 페이지네이션하여 조회합니다. 무한스크롤 방식을 지원합니다.

### 📌 엔드포인트

`GET /v1/admin/scores/members/{memberId}/activity-records`

### 🔐 인증

- **인증 필요 여부:** JWT 인증 필요 (관리자 권한)

> 요청 헤더(Header)에 아래와 같이 Authorization 필드를 포함해야 합니다.
>
> `Authorization: Bearer {JWT_TOKEN}`

### ▶️ 요청 (Request)

<aside>

### Headers

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `Authorization` | String | `Bearer {JWT_TOKEN}` | ✅ |

### Path Parameters

| Key | Type | 설명 |
| --- | --- | --- |
| `memberId` | Long | 조회할 회원의 고유 ID |

### Query Parameters

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `pageNum` | int | 페이지 번호 (0부터 시작) | ✅ |
| `pageSize` | int | 한 페이지에 보여줄 아이템 개수 | ✅ |

### ▶️요청 예시

```
GET /v1/admin/scores/members/1/activity-records?pageNum=0&pageSize=10
```

</aside>

---

---

### ◀️ 응답 (Response)

<aside>

### 성공 (Success)

| HTTP Status | 의미 |
| --- | --- |
| `200 OK` | 활동기록 조회 성공 |

**Body**

| Key | Type | 설명 |
| --- | --- | --- |
| `code` | int | HTTP 상태 코드 |
| `message` | String | 응답 메시지 |
| `data.content` | Array | 활동기록 목록 |
| `data.content[].activityRecordId` | String | 활동기록 ID (TSID, 문자열로 직렬화) |
| `data.content[].activityType` | String | 활동 유형 코드 (예: "ATTENDANCE") |
| `data.content[].activityName` | String | 활동 유형 한글명 (예: "정규세션 출석") |
| `data.content[].scoreType` | String | 점수 유형 ("REWARD" 또는 "PENALTY") |
| `data.content[].activityDate` | String | 활동일자 (yyyy-MM-dd) |
| `data.content[].appliedScore` | BigDecimal | 적용된 점수 |
| `data.pageNumber` | int | 현재 페이지 번호 |
| `data.pageSize` | int | 페이지 크기 |
| `data.numberOfElements` | int | 현재 페이지의 요소 수 |
| `data.isLast` | boolean | 마지막 페이지 여부 |

### 응답 예시

```json
{
  "code": 200,
  "message": "[활동 기록]을 조회했습니다.",
  "data": {
    "content": [
      {
        "activityRecordId": "394829301948293019",
        "activityType": "ATTENDANCE",
        "activityName": "정규세션 출석",
        "scoreType": "REWARD",
        "activityDate": "2025-03-15",
        "appliedScore": 2.0
      }
    ],
    "pageNumber": 0,
    "pageSize": 10,
    "numberOfElements": 1,
    "isLast": true
  }
}
```

</aside>

### 실패 (Error)

<aside>

| HTTP Status | 의미 | 에러 코드 |
| --- | --- | --- |
| `401 Unauthorized` | 인증 실패 | `UNAUTHORIZED` |
| `403 Forbidden` | 관리자 권한 필요 | `FORBIDDEN` |

**응답 예시 (401 Unauthorized)**

```json
{
  "code": 401,
  "message": "인증이 필요합니다."
}
```

</aside>
