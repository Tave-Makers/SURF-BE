## 활동기록 수정

### 개요

특정 활동기록의 활동유형(activityType)과 활동일자(activityDate)를 수정합니다.

### 📌 엔드포인트

`PATCH /v1/admin/activity-records/{activityRecordId}`

### 🔐 인증

- **인증 필요 여부:** JWT 인증 필요 (관리자 권한)

> 요청 헤더(Header)에 아래와 같이 Authorization 필드를 포함해야 합니다.
>
> `Authorization: Bearer {JWT_TOKEN}`

### ▶️ 요청 (Request)

<aside>

### Headers

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `Content-Type` | String | `application/json` | ✅ |
| `Authorization` | String | `Bearer {JWT_TOKEN}` | ✅ |

### Path Parameters

| Key | Type | 설명 |
| --- | --- | --- |
| `activityRecordId` | Long | 수정할 활동기록의 고유 ID |

### Body

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `activityType` | String | 변경할 활동 유형 (예: "ATTENDANCE") | ⬜️ |
| `activityDate` | String | 변경할 활동일자 (yyyy-MM-dd) | ⬜️ |

### ▶️요청 예시

```json
{
  "activityType": "LATE_ATTENDANCE",
  "activityDate": "2025-03-20"
}
```

</aside>

### ◀️ 응답 (Response)

<aside>

### 성공 (Success)

| HTTP Status | 의미 |
| --- | --- |
| `200 OK` | 활동기록 수정 성공 |

**Body**

| Key | Type | 설명 |
| --- | --- | --- |
| `code` | int | HTTP 상태 코드 |
| `message` | String | 응답 메시지 |
| `data` | null | 응답 데이터 없음 |

### 응답 예시

```json
{
  "code": 200,
  "message": "[활동 기록]을 수정했습니다.",
  "data": null
}
```

</aside>

---

### 실패 (Error)

<aside>

| HTTP Status | 의미 | 에러 코드 |
| --- | --- | --- |
| `400 Bad Request` | 이미 삭제된 활동기록 | `ACTIVITY_RECORD_ALREADY_DELETED` |
| `401 Unauthorized` | 인증 실패 | `UNAUTHORIZED` |
| `403 Forbidden` | 관리자 권한 필요 | `FORBIDDEN` |
| `404 Not Found` | 존재하지 않는 활동기록 | `ACTIVITY_RECORD_NOT_FOUND` |

**응답 예시 (404 Not Found)**

```json
{
  "code": 404,
  "message": "존재하지 않는 [활동기록]입니다."
}
```

**응답 예시 (400 Bad Request)**

```json
{
  "code": 400,
  "message": "이미 삭제된 [활동기록]입니다."
}
```

</aside>
