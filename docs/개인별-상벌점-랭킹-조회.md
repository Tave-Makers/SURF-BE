## 개인별 상벌점 랭킹 조회

### 개요

개인별 상벌점 현황을 totalScore 내림차순으로 조회합니다. 무한스크롤 방식의 페이지네이션을 지원합니다.

### 📌 엔드포인트

`GET /v1/admin/scores/members`

### 🔐 인증

- **인증 필요 여부:** JWT 인증 필요 (관리자 권한)

> 요청 헤더(Header)에 아래와 같이 Authorization 필드를 포함해야 합니다.
>
> `Authorization: Bearer {JWT_TOKEN}`

### ▶️ 요청 (Request)

<aside>

### Headers

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `Authorization` | String | `Bearer {JWT_TOKEN}` | ✅ |

### Query Parameters

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `pageNum` | int | 페이지 번호 (0부터 시작) | ✅ |
| `pageSize` | int | 한 페이지에 보여줄 아이템 개수 | ✅ |

### ▶️요청 예시

```
GET /v1/admin/scores/members?pageNum=0&pageSize=10
```

</aside>

---

### ◀️ 응답 (Response)

<aside>

### 성공 (Success)

| HTTP Status | 의미 |
| --- | --- |
| `200 OK` | 개인별 상/벌점 현황 조회 성공 |

**Body**

| Key | Type | 설명 |
| --- | --- | --- |
| `code` | int | HTTP 상태 코드 |
| `message` | String | 응답 메시지 |
| `data.content` | Array | 개인별 상벌점 목록 |
| `data.content[].memberId` | Long | 회원 ID |
| `data.content[].profileImageUrl` | String | 프로필 이미지 URL |
| `data.content[].name` | String | 회원 이름 |
| `data.content[].part` | String | 파트명 (예: "백엔드", "프론트엔드") |
| `data.content[].rewardTotal` | BigDecimal | 상점 합계 |
| `data.content[].penaltyTotal` | BigDecimal | 벌점 합계 (양수) |
| `data.content[].totalScore` | BigDecimal | 총 점수 |
| `data.pageNumber` | int | 현재 페이지 번호 |
| `data.pageSize` | int | 페이지 크기 |
| `data.numberOfElements` | int | 현재 페이지의 요소 수 |
| `data.isLast` | boolean | 마지막 페이지 여부 |

### 응답 예시

```json
{
  "code": 200,

    "pageSize": 10,

    "isLast": true
  }
}
```

</aside>

### 실패 (Error)

<aside>

| HTTP Status | 의미 | 에러 코드 |
| --- | --- | --- |
| `401 Unauthorized` | 인증 실패 | `UNAUTHORIZED` |
| `403 Forbidden` | 관리자 권한 필요 | `FORBIDDEN` |

**응답 예시 (401 Unauthorized)**

```json
{
  "code": 401,
  "message": "인증이 필요합니다."
}
```

</aside>
