## 약관 동의

### 개요

로그인한 회원이 서비스 약관 동의 여부를 `true`로 변경하는 API입니다. 이미 동의한 회원이 다시 호출해도 동일하게 성공 처리되는 멱등 API입니다.

### 📌 엔드포인트

`PATCH /v1/user/members/terms/agree`

> 예시: PATCH /v1/user/members/terms/agree

### 🔐 인증

- **인증 필요 여부:** JWT 인증 필요

> 요청 헤더(Header)에 아래와 같이 Authorization 필드를 포함해야 합니다.
>
> `Authorization: Bearer {JWT_TOKEN}`

### ▶️ 요청 (Request)

<aside>

### Headers

| Key | Type | 설명 | 필수 |
| --- | --- | --- | --- |
| `Authorization` | String | `Bearer {JWT_TOKEN}` | ✅ |

### Path Parameters

없음

### Query Parameters

없음

### Body

요청 본문 없음

### ▶️요청 예시

```http
PATCH /v1/user/members/terms/agree HTTP/1.1
Authorization: Bearer {JWT_TOKEN}
```

</aside>

---

---

### ◀️ 응답 (Response)

<aside>

### 성공 (Success)

| HTTP Status | 의미 |
| --- | --- |
| `200 OK` | 약관 동의 처리 성공 |

**Body**

| Key | Type | 설명 |
| --- | --- | --- |
| `code` | int | HTTP 상태 코드 |
| `message` | String | 응답 메시지 |
| `data` | null | 응답 데이터 없음 |

### 응답 예시

```json
{
  "code": 200,
  "message": "[약관 동의]를 완료했습니다.",
  "data": null
}
```

</aside>

---

### 실패 (Error)

<aside>

| HTTP Status | 의미 | 에러 코드 |
| --- | --- | --- |
| `401 Unauthorized` | 유효하지 않거나 만료된 JWT | `UNAUTHORIZED` |
| `403 Forbidden` | 인증 토큰이 없거나 회원 권한이 없음 | `FORBIDDEN` |
| `404 Not Found` | 인증된 회원 정보를 찾을 수 없음 | `MEMBER_NOT_FOUND` |
| `500 Internal Server Error` | 서버 내부 오류 | `INTERNAL_SERVER_ERROR` |

**응답 예시 (401 Unauthorized)**

```json
{
  "message": "Invalid or expired access token"
}
```

**응답 예시 (404 Not Found)**

```json
{
  "code": 404,
  "message": "존재하지 않는 [회원]입니다.",
  "data": null
}
```

</aside>
